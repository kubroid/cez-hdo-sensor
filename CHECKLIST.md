# ✅ Контрольный список для публикации

Прежде чем добавлять репозиторий в HACS, убедитесь:

## 📋 GitHub:
- [ ] Репозиторий публичный
- [ ] Создан релиз с тегом (например, v1.1.0)
- [ ] Файлы загружены в ветку main/master
- [ ] hacs.json в корне репозитория
- [ ] custom_components/cez_hdo/ со всеми файлами

## 🔧 HACS:
- [ ] HACS установлен и работает
- [ ] Вы в разделе HACS → Integrations (НЕ в Настройках → Дополнения!)
- [ ] Выбрана категория "Integration" (НЕ "Add-on"!)
- [ ] URL точный: `https://github.com/kubroid/cez-hdo-sensor`

## 🚨 Частые ошибки:

### "not a valid add-on repository"
**Причина**: Добавляете в Настройки → Дополнения вместо HACS → Integrations
**Решение**: Идите в HACS → Integrations

### "Repository structure for v1.1.0 is not compliant"
**Причина**: Нет релиза или неправильная структура
**Решение**: Создайте релиз в GitHub

### Репозиторий не найден
**Причина**: Опечатка в URL или репозиторий приватный
**Решение**: Проверьте URL и сделайте репозиторий публичным

## 🎯 После успешного добавления:

1. Найдите "CEZ HDO Sensor" в списке интеграций HACS
2. Нажмите "Download"
3. Перезапустите Home Assistant
4. Добавьте интеграцию через Settings → Devices & Services

## 📞 Если ничего не помогает:

1. Проверьте логи Home Assistant на ошибки
2. Попробуйте ручную установку
3. Создайте issue в GitHub с подробным описанием проблемы
